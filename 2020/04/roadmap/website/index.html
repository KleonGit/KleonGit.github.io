网站入门路线地图2020

发布于 四月 18, 2020    更新于 四月 19, 2020

一份网站搭建入门路线图，标记了关键节点与参考资源。

我在入门网站搭建的时候，参考了不少资料，包括github上的全栈入门的roadmap，比如developer-roadmap。

我会主要参考这个roadmap，并根据我的学习经验加以补充。

前置技能

前置技能基本上是编程必备了，但这些所谓的前置技能往前还有前置，就是一般的编程基础，可以另开其他roadmap。

Git

git可谓是目前代码版本管理，协作开发必备了，也是由Linux系统作者Linus写的。

和其他工具一样，学习git不需要在一开始掌握多少命令，简单掌握基本的操作，以后随着项目规模自然学习新的命令即可，也就是遵循最小化原则。

  • 手把手git教程
  • git documentation

Basic Terminal Usage

终端，或者叫命令行，是和计算机的最轻量交互接口。相比于GUI需要鼠标精确点击按钮，在终端里，可以通过命令与脚本实现自动化操作。对于前端来说，学会基本的终端操作，能够启动前端服务程序就可以继续了；对于后端（Backend）和运维（DevOps）来说，GUI是没有必要的，用于复杂图形显示的大量计算资源可以节省下来用作服务算力。由于历史原因，服务器目前一般使用linux操作系统，后端（Backend）和运维（DevOps）就需要对终端操作更熟悉，之后再结合。

Windows上推荐使用Power Shell，或者用安装完git之后自带的git bash（适合习惯linux命令的人），推荐使用chocolatey包管理器。
Linux/Mac OS上推荐使用zsh搭配Oh-My-Zsh。

  • 鸟哥的私房菜

Linux的经典中文资源，申请云服务器（现在申请云服务器很方便，学习用按量付费也很便宜，而且不用担心搞坏电脑或者电脑配置不够）或者安装Windows Linux Subsystem的看第一部分前三节和最后一节、第二部分和第三部分，本地安装（多系统或者虚拟机）的看前三部分。

Data Structures & Algorithms

数据结构和算法，不是必要的，学习编程语言、解决具体问题时会接触到。

Github

既然提到git，自然github也是必须要提的。作为全球最大开源代码分享协作平台，可以从不少优秀项目中学习。

Licenses

如果要在其他项目中使用别人的开源代码，自然要注意版权问题，也就是确认每个开源项目使用的授权许可证。

  • 七种开源许可协议

Semantic Versioning 版本号

在开发过程中，需要用到其他人开发的软件包，这些软件包可能不断开发升级，API和功能持续变动，因此需要在自己的项目中指定依赖的包版本。

  • 语义化版本控制规范

SSH 远程登录

主要用于登录linux主机，PowerShell自带ssh，在Windows上可能比putty好用。

HTTP/HTTPS and APIs

你需要先了解计算机网络的基本原理，不必深入了解，大致对基本的运作方式有个形象的理解，在随后的使用中加深了解即可。

HTTP、HTTPS是服务器和浏览器用来传输网页的内容的协议，约定了传输格式以及加密方式。

API（Application Programming Interface）在这里指的是一个网页地址，但和一般网页地址不同的是，可以通过API完成对后台数据的增加、查询、更新、删除（Create Read Update Delete, CRUD）或者调用后台程序，因此，API就相当于后台程序的编程接口。

  • 互联网是怎样工作的
  • HTTP教程
  • 图解HTTP
  • 计算机网络-自顶向下方法

Design Patterns

设计模式，不是必要的，学习编程语言、解决具体问题时会接触到。

Character Encodings

字符编码，貌似很小的东西，但是真的折腾人，比如中文编码。一般认准utf-8。

  • 字符编码：ASCII、Unicode、UTF-8

分支

恭喜你，在掌握了网络原理和软件开发基础后，你可以选择接下来的学习内容：

前端

以javascript、html、css为三大武器，构建视觉交互界面，俗称写网页。在框架的助力下，写一个简易可用的网页不再麻烦。如果只需要静态网页——也就是只有前端展示交互，不需要和后端交换数据的网页，或者有其他API可以调用，那么学习前端就足够了。

后端

多种多样，千变万化，以服务为基本单元，通过API与前端沟通，使用数据库提高数据存储操作的效率。除此之外，后端可以增加各种服务：视频分发，图像检测，语音识别，语义分析，信息搜索等等，进而组合成更复杂的具体应用：商品推荐，医疗辅助，直播带货，网络游戏等等。
可见后端服务内容之丰富，每一项服务背后都是一个团队的积累。这个roadmap中的后端，仅仅涵盖了一般的后端功能，比如：数据存取、检索，API，部署运维。
