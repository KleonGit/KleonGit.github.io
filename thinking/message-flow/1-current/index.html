消息流-1 在各个应用之间

发布于 四月 8, 2021    更新于 四月 8, 2021

消息是信息传递的最基本单位。

声音，图像，文本，一条消息自身具备独立的语义。

各式各样的应用使用数据库把消息按自己的格式组织起来。

现状

备忘录

大概只会看一次而怕自己忘掉的

很多串文本

记录密码，书名，摘抄，灵感，还想放下图片，文件，音频

代办事项

一些早晚忘了更新状态的

一个check复选框 + 一串文本 + 一个定时提醒器

按时间，按位置反复提醒

收藏夹

一些你以为你会再用到的

一串文本 + 一串网页链接

项目计划

比代办事项复杂一点的甩锅给很多人的

一串文本 + 一些自定义属性

文本之前加上了母女关系，能按树状图排列

可以按自定义属性中的一项按看板排列，通常因为没有大白板和便签纸

文档

爷爷是Word

不知道是写给谁看的但至少自己会看一遍的

很多串文本 + 假装表格的文本 + 任何文件链接 + 复杂的页面布局

哦万年不更新所以经常会有坑

可以当成完全展开的专家系统用

文献引用好麻烦

表格

爷爷是Excel

看似有无限空间然而只用了左上角的

一张数据表 + 到处填公式的单元格 + 放不下的图表

除了煮咖啡什么都能做

幻灯片

爷爷是PowerPoint，大名PPT

把文档的大段文字解释去掉再切成片的

文档 + 从Excel粘来的表格和图表 + 从网上粘来的图片和图标

尺寸可以非常大

还可以加动画，只要我想，全写在一张上也没有问题，导出pdf自带加密

办公IM

大部分人都是观众唯一的参与感是收到OK的表情的

无限长 普通文本串 / anything外链

想整合文档，表格，PPT，文件盘，webhook

老板在的群都置顶有新消息就要爬楼看看有没有提到自己

已读不回

生活IM

在吗吃啥玩啥到了吗哈哈哈哈哈的

无限长 普通文本串 / 图片(jpg,gif)

经常给自己发文件

大部分群都屏蔽就是不退没事还要去爬楼吃瓜

笔记应用

一种不知道为什么特殊的文档

白纸很大无标题前十分钟删除键频率高于50%

迁移超麻烦

算了我还是用markdown放github吧

数据库

听起来很难用还要学SQL实际上技术工作大概可能会用到

调查问卷

就是Form

一种可以交互的文档

文本 + 按钮 + 各种组件

有一个提交按钮要按

归纳

文本

最基本的元素，无处不在

样式

包括文本样式，页面布局

可以很灵活，但通常自己选的很丑，不如用系统默认

数据

存在数据表里

不用SQL还可以：表格，看板，卡片列表

链接

认识的就预览不认识的就留着，包括一切非文本

控件

可以和文本各司其职和数据表关联即可

节奏感

分割线帮你切出幻灯片

自动化

快速复用低代码
